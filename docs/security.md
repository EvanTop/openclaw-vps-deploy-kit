# 安全建议（远端常驻必看）

> 目标：让你在 VPS 上运行 OpenClaw 时，尽量避免“公网裸奔 + 被扫到 + 被撞库”的风险。

官方参考（强烈建议读）：
- https://docs.openclaw.ai/gateway/remote

---

## 1) 最安全默认（推荐所有人）

- Gateway **绑定 loopback（127.0.0.1）**
- 远程访问走：
  - SSH 隧道（通用兜底）
  - 或 Tailscale（长期方案）

这样公网扫描不到你的 Gateway 端口。

---

## 2) 不建议做的事

- 直接在安全组/防火墙里开放 18789 给全网
- 直接把 ws:// 暴露在公网

---

## 3) 如果你“必须”公网暴露（只给高级用户）

你至少要做到：

1) **启用认证（token/password）**
2) **限制来源 IP**（只允许你/客户的固定 IP 或办公网出口）
3) **优先 TLS（wss）**（例如通过可信反向代理）
4) **定期轮换凭证**（尤其是交付给客户后）

---

## 4) SSH 安全建议

- 禁用密码登录，启用密钥登录
- 改默认 22 端口（可选，但有帮助）
- 开启 fail2ban（可选）

---

## 5) 交付条款建议（做服务必备）

- 明确：服务方不长期保管客户 API Key（除非客户明确授权并采用密码管理器共享）
- 明确：如客户要求公网暴露，需签风险确认
